# Tales from the Дед — Интерактив & Персонализация: Идеи и Roadmap

> **Философия:** Пользователь — не гость. Он жертва. Или палач. Или оба.  
> **Тон:** Чёрный юмор, деревенская безнадёга, крипота, ужастики. Дед всегда на шаг впереди.  
> **Ключевой принцип:** Любой интерактив должен заставлять пользователя почувствовать себя НЕ в безопасности.

---

## 🏗️ Уровни реализации

| Уровень | Инфраструктура | Сложность | Данные |
|---|---|---|---|
| **L1** | GitHub Pages (статика) | Низкая | localStorage |
| **L2** | VPS Lite (API + SQLite) | Средняя | SQLite, простой REST |
| **L3** | VPS Full (БД + WebSocket) | Высокая | PostgreSQL, Redis, auth |

## 🪦 L1: GitHub Pages — Кладбище, Проклятия, Карта (без бэкенда)

### 1. Персональное кладбище (`cemetery.html`)

**Концепция:** У каждого пользователя — своё кладбище. Каждый визит — новая могилка. Чем больше визитов, тем больше кладбище разрастается.

**Механика:**
- При первом визите — одна могила с именем пользователя и датой "смерти" (текущая дата).
- При повторных визитах — новые могилки рядом. Дед "хоронит" новых "гостей" (NPC).
- fearLevel влияет на вид: 
  - 0-1: заброшенное, трава, паутина
  - 2-3: туман, свечи, имена на камнях
  - 4-5: кровь, треснувшие камни, анимации призраков, имя пользователя на главном кресте

**Чёрный юмор:**
- Эпитафии генерируются случайно: *"Здесь лежит [Имя]. Пришёл посмотреть — остался навсегда."*
- *"Василий. Покойся с миром. Хотя Дед не обещает."*
- *"Неизвестный гость. Дед забыл имя, но не забыл лицо."*
- Могилка с крестом вверх ногами — если пользователь заходил ночью (проверка `new Date().getHours()`).

**Технически:**
- SVG/CSS-генерация кладбища. localStorage: `ded_cemetery_visits`, `ded_cemetery_epitaphs[]`.
- Canvas для тумана/крови (опционально, CSS анимации хватит).

**Приоритет:** 🔥 ВЫСОКИЙ (реализуется на статике, максимальный эффект).

### 2. Интерактивная карта деревни (`map.html`)

**Концепция:** Карта деревни с домами персонажей. Клик — заглянуть в окно. Ночью — другая картина.

**Объекты на карте:**
- **Изба Деда** — вход в погреб (ссылка на `cellar.html` или будущую пыточную). Дым из трубы. Ночью — красный свет в окнах.
- **Дом Коли** — свет горит. Если кликнуть — тень за окном. Клик 3 раза — Коля открывает окно и шепчет что-то (пасхалка).
- **Дом Алёнки** — музыка тихо играет. Клик — окно захлопывается, смех.
- **Колодец** — клик "бросить монетку" (анимация). Выпадает предсказание от Деда: *"Ты умрёшь... от скуки, если не зайдёшь завтра."*
- **Баня** — пар. Ночью — кровь на снегу рядом.
- **Погреб (отдельный)** — заперто. Нужен ключ (пасхалка где-то на сайте).

**Чёрный юмор:**
- Предсказания колодца: *"Сегодня твой счастливый день. Завтра — нет."*, *"Не выходи из дома. Хотя ты уже здесь."*
- При наведении на дом — появляется текст: *"Свет горит. Кто-то не спит. Кто-то ждёт."*

**Технически:**
- CSS + SVG карта. Один большой SVG или CSS grid с абсолютным позиционированием.
- localStorage: `ded_well_predictions[]`, `ded_map_clicks`.

**Приоритет:** 🔥 ВЫСОКИЙ (даёт ощущение "мира", куда можно возвращаться).

### 3. Система проклятий (`curse-system.js`)

**Концепция:** Пользователь случайно получает "проклятие" при определённых действиях. Сайт начинает вести себя... иначе.

**Виды проклятий:**
- **"Глаза Деда"** — в каждом углу экрана появляются глаза, следящие за курсором (всегда, не только idle).
- **"Скрипучие буквы"** — текст начинает дрожать, случайные буквы заменяются на другие. Проходит через 10 секунд после клика.
- **"Кровавый след"** — за курсором остаётся след из капель крови, который исчезает через 5 секунд.
- **"Зеркальный мир"** — весь сайт переворачивается по горизонтали на 3 секунды.
- **"Дед говорит"** — рандомный текст на странице заменяется на реплику Деда: *"Я вижу, что ты читаешь."*

**Как получить:**
- Случайно (5% при каждом клике по карточке персонажа).
- При повторном визите после полуночи (localStorage + `Date`).
- При попытке уйти со страницы (`beforeunload` — *"Дед не отпускает так просто."`).

**Как снять:**
- Провести на сайте 5 минут.
- Найти "святую воду" (клик по спрятанному кресту где-то в DOM).
- Или... никак. Дед решает, когда отпустить.

**Технически:**
- CSS классы + JS таймеры. localStorage: `ded_curses[]`.

**Приоритет:** 🔥 ВЫСОКИЙ (легко, эффектно, создаёт стресс).

### 4. Гостевая книга проклятий (GitHub Issues / Discussions)

**Концепция:** Не обычная гостевая, а "Стена пожеланий Деду".

**Механика:**
- Пользователь пишет имя + сообщение. 
- Дед "отвечает" заранее заготовленными фразами (рандом из массива).
- Ответы злые, но смешные: *"Ты хочешь, чтобы я перестал? А я хочу, чтобы ты перестал дышать. Пока ничья."*

**Реализация:**
- GitHub Issues API — создавать issue как "запись в гостевой".
- Или просто GitHub Discussions.
- Пользователь не видит Issues, видит только "Стену" через GitHub API (read-only).
- Дед "модерирует" = владелец repo закрывает issue с комментарием.

**Приоритет:** 🟡 СРЕДНИЙ (требует GitHub API, но не требует VPS).

## 🔥 L2: VPS Lite — API + SQLite (Node.js/Python)

### 7. Пыточная / Погреб (`cellar.html` + API)

**Концепция:** Пользователь может "сажать" персонажей в погреб и "работать" с ними. Это не жестокость — это чёрный юмор в стиле "Дед знает, что ты сделал".

**Механика:**
- Пользователь выбирает персонажа из списка (Коля, Алёнка, Мила, Витек, Куролес, Михайловна, Дед).
- Дед "приводит" персонажа в погреб. Анимация: тень тащит фигуру в дверь.
- Действия (кнопки):
  - **"Дать самогона"** — персонаж расслабляется, начинает говорить. Может раскрыть секрет.
  - **"Показать ложку"** — персонаж пугается, но не говорит. +fearLevel.
  - **"Поговорить"** — рандомная реплика. Повторяется, пока не дать самогона.
  - **"Отпустить"** — персонаж уходит, но запоминает пользователя. При следующем визите — другая реакция.
- **Состояние персонажа:** мораль (0-100), пьяность (0-100), количество секретов раскрыто.
- **Секреты:** каждый персонаж знает 3-5 секретов о вселенной. Раскрываются случайно при действиях.

**Чёрный юмор:**
- Коля: *"Дай самогона, я всё расскажу! Дед козёл, но это между нами."*
- Алёнка: *"Ты думаешь, я боюсь? Я Витека боюсь больше. Он после бани..."*
- Мила: *"Я не буду ничего говорить. ...Хотя, если ты Деда спросишь про колодец..."*
- Витек: *"Ложка? Я ложку ел, когда ты ещё молоко на губах не обсох. Дай лучше закусить."*
- Куролес: *"Пытать? Ты меня? Хаха! Я уже 20 лет в этом погребе! Дед меня не выгоняет — он боится."*
- Михайловна: *"Молодой человек, я вам не расскажу ничего. ...Хотя, если вы знаете, что Дед делает с курами..."*
- Дед: *"Ты думаешь, ты меня пытаешь? Я тебя пытаю. Уже 10 минут."*

**Технически:**
- SQLite таблица: `users`, `prisoners`, `secrets`, `interactions`.
- API: `POST /api/cellar/visit`, `POST /api/cellar/action`, `GET /api/cellar/state`.
- Auth: UUID или simple token (localStorage → передаётся в заголовке).
- VPS: облачный сервер (cloud.ru), минимальный Node.js/Express + SQLite.

**Приоритет:** 🔥🔥 ОЧЕНЬ ВЫСОКИЙ (флагманская фича, уникальность).

### 8. Личный кабинет Деда (`dashboard.html` + API)

**Концепция:** Каждый пользователь — "подопытный" Деда. У него есть личный дашборд.

**Что видит пользователь:**
- **Статистика:** количество визитов, общее время на сайте, количество проклятий, достижения.
- **Кладбище:** SVG-рендер всех "могил" (визитов), даты, эпитафии.
- **Пыточная:** кто сейчас в погребе, состояние, раскрытые секреты.
- **Досье:** Дед собрал о пользователе — IP, браузер, ОС, разрешение, город, визиты, имя, страх.
- **Почта от Деда:** список "писем" (предгенерированных сообщений с датами).
- **Достижения:** грид с иконками.

**Чёрный юмор:**
- Дашборд называется "Личное дело №[random]".
- Аватар пользователя — чёрный крест или силуэт.
- Статус: *"Статус: жив. Пока."* / *"Уровень страха: [fearLevel]. Рекомендация: не спать."*
- Письма: *"Я заметил, что ты не заходил 3 дня. Ты думал, я забыл? Нет. Я ждал."*

**Технически:**
- API: `GET /api/user/{uuid}/dashboard`.
- SQLite: `users` (uuid, name, city, visits, total_time, fear_level, created_at).
- Frontend: `dashboard.html` — fetch к API, рендеринг.

**Приоритет:** 🔥 ВЫСОКИЙ (центр вселенной пользователя).

### 9. Почта от Деда (Email / Telegram Bot)

**Концепция:** Дед пишет пользователю, если тот не заходил N дней.

**Триггеры:**
- 3 дня без визита: *"Ты куда пропал? Я начал скучать. Это плохо. Для тебя."*
- 7 дней: *"Неделя. Я уже выкопал могилку. На всякий случай. Размер примерно твой."*
- 30 дней: *"Месяц. Я забыл, как ты выглядишь. Но не забыл, как ты пах. Приходи — напомни."*
- После пытки персонажа: *"Коля рассказал много интересного. О тебе. Приходи, обсудим."*

**Реализация:**
- Email: SendGrid / Mailgun (нужен домен, но можно и бесплатный tier).
- Telegram Bot: пользователь подписывается на бота, получает сообщения.
- VPS: cron job, проверяет `last_visit`, отправляет через API.

**Приоритет:** 🟡 СРЕДНИЙ (вовлечение, но требует внешних сервисов).

### 10. Синхронизация между устройствами

**Концепция:** Пользователь заходит с телефона — продолжает пытать Коля, начатого на десктопе.

**Механика:**
- При первом визите генерируется UUID (`crypto.randomUUID()`), сохраняется в localStorage.
- При вводе имени/города — отправка на API `POST /api/user/register` с UUID.
- Дальше всё состояние хранится на VPS.
- При заходе с нового устройства — запрос `GET /api/user/{uuid}` → восстановление localStorage.

**Технически:**
- UUID как primary key. Никаких паролей — Дед не заморачивается с безопасностью жертв.

**Приоритет:** 🟡 СРЕДНИЙ (удобство, но не критично).

## 🚀 L3: VPS Full — Полноценный бэкенд

### 11. Персонажи "живут" в реальном времени (Real-time)

**Концепция:** Даже когда пользователя нет, персонажи в погребе двигаются, болтают, Дед ходит.

**Механика:**
- VPS: background worker (cron или setInterval), каждые 5 минут обновляет состояние.
- Персонажи в погребе: мораль падает со временем, пьяность уходит.
- Если пользователь не заходил 3 дня — персонаж "сбежал" или "Дед его выгнал".
- При заходе — Дед комментирует: *"Коля ушёл вчера. Сказал, что ты тряпка. Я с ним согласен."*

**Технически:**
- PostgreSQL + Redis (кеш состояния). Или просто SQLite + cron.
- Node.js worker process.

**Приоритет:** 🟢 НИЗКИЙ (круто, но сложно и ресурсоёмко).

### 12. Голосование / Коллективный выбор

**Концепция:** Пользователи голосуют, что происходит дальше.

**Механика:**
- "Кого пытать следующим?" — голосование на неделю.
- "Что сделать с Колей?" — отпустить / дать ещё самогона / показать вилку.
- "Какой эпизод вы хотите увидеть?" — выбор из 3 synopsis.
- Результаты влияют на сюжет комикса.

**Технически:**
- API: `POST /api/vote`, `GET /api/vote/results`.
- SQLite: `votes` таблица.
- Защита: UUID может голосовать 1 раз.

**Приоритет:** 🟡 СРЕДНИЙ (вовлечение аудитории).

---

### 13. WebSocket — "Прямой эфир из деревни"

**Концепция:** На странице — чат/лог, куда Дед "пишет" в реальном времени.

**Механика:**
- WebSocket соединение. Каждые 30 секунд — рандомное событие:
  - *"Дед вышел покурить."*
  - *"Из погреба доносится крик. Похоже, Коля нашёл паука."*
  - *"Алёнка пытается убежать. Дед уже идёт."*
  - *"Кто-то стучит в окно. Не обращай внимания."*
- Пользователь может "ответить" Деду (ввод текста, но ответы рандомные).

**Технически:**
- WebSocket сервер (Node.js `ws` или Socket.io).
- Не требует auth — просто broadcast.

**Приоритет:** 🟢 НИЗКИЙ (фан, но не обязательно).

---

### 14. Telegram Mini App / Web App

**Концепция:** Весь сайт — как Telegram Mini App. Пользователь открывает в Telegram, авторизация мгновенная.

**Плюсы:**
- Не нужен отдельный auth — Telegram даёт `initData` с user_id.
- Push-уведомления через Telegram Bot.
- Вирусный рост — шеринг в чатах.

**Механика:**
- Пользователь открывает бота → "Войти в деревню" → Web App открывается.
- Все данные синхронизированы с VPS.
- Дед пишет в личку Telegram, если не заходил.

**Приоритет:** 🟡 СРЕДНИЙ (аудитория, но требует Telegram Bot + Mini App wrapper).

## 💡 Технологический стек для VPS

| Компонент | Технология | Почему |
|---|---|---|
| **VPS** | cloud.ru (или любой) | Уже есть опыт, российский провайдер |
| **Backend** | Node.js + Express | Просто, быстро, много библиотек |
| **БД** | SQLite (L2) → PostgreSQL (L3) | SQLite для старта, Postgres для роста |
| **Auth** | UUID + localStorage | Никаких паролей — Дед не любит сложности |
| **Email** | SendGrid / Mailgun / SMTP | Бесплатный tier |
| **Telegram** | node-telegram-bot-api | Push + Mini App |
| **WebSocket** | ws (npm) | Лёгкий, без лишнего |
| **Deploy** | PM2 + Nginx | Проверенная связка |
| **Frontend** | Тот же HTML/CSS/JS | API fetch + DOM |
